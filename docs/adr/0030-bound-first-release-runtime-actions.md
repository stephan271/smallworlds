# Bound first-release in-cluster actions

The first in-cluster Operator Console is observe, configure, and propose oriented. Its Runtime Actions are limited to GitOps proposals, backup configuration and validation, Operator Device enrollment management, observation refresh, Setup Journey credential rotation, and planned retry of idempotent setup work. Workload restarts, forced Argo sync, resizing, database repair, restoration, resource deletion, and arbitrary Kubernetes mutation remain separately designed future capabilities rather than generic escape hatches.
