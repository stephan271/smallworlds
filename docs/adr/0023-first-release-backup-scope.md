# Configure and validate backups before adding restore operations

The first-release Backups section will show dataset coverage, local and offsite Recovery Points, retention, freshness, recoverability windows, and the most recent Restore Drill. It will guide offsite S3 configuration, store the resulting Cluster Secret, verify access and versioning, and run one bounded backup/replication validation as setup evidence. Restore execution, recovery-point deletion, and retention mutation remain explicit future capabilities; their roadmap may be shown, but the UI will not present inactive controls as usable operations.
