# Nextcloud Tenant (`infrastructure/kubernetes/tenants/nextcloud/*.yaml`)

Nextcloud provides file storage and synchronization. Its configuration connects it heavily to Garage (for primary storage), CloudNativePG (database), and Keycloak (Identity).

## Key Infrastructure Integrations

### 1. Primary Storage via S3 (`values.yaml`)
Instead of relying on a standard Kubernetes PersistentVolume (PVC) to store user files, Nextcloud is configured to use **Garage S3 as its primary object storage**.
- **Object Store Config**: The `objectStore` section maps Nextcloud to `garage.garage-system.svc.cluster.local:3900`.
- **Credentials**: It loads `OBJECTSTORE_S3_KEY` and `OBJECTSTORE_S3_SECRET` directly from the `garage-secret` provisioned by the base job. This approach allows the storage backend to scale horizontally independent of the Nextcloud pods.

### 2. OIDC Configuration via OCC (`oidc-config-job.yaml`)
Nextcloud doesn't provide a fully declarative way to configure OIDC through environment variables.
- **Pod Exec**: The `oidc-config-job.yaml` is a Sync hook that waits for Nextcloud to install itself, then uses `kubectl exec` to run `php occ` commands directly inside the Nextcloud container.
- **Setup**: It installs the `user_oidc` app and configures it to point to the Keycloak discovery URL. The client secret is sourced from the `keycloak-secret` generated by the `keycloak-client-job` base.
- **Auto-Redirect**: Critically, it runs `php occ config:app:set user_oidc allow_multiple_user_backends --value=0`. This tells Nextcloud to disable its native login form and instantly redirect unauthenticated users to the Keycloak SSO portal.

### 3. Proxy Configuration (`values.yaml`)
- **Trusted Proxies**: In the `configs` section, a custom PHP configuration snippet adds `10.42.0.0/16` and `10.43.0.0/16` (the Kubernetes internal pod/service subnets) to the `trusted_proxies` list. Without this, Nextcloud would reject logins and fail to generate correct redirect URLs behind the Traefik ingress controller.
