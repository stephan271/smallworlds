# Project Rules and Guidelines

## Keycloak Client Registration Secret Contract

Any tenant application deploying OIDC client registration via the shared job base (`../../bases/keycloak-client-job`) MUST retrieve its credentials from the standard namespace-local secret and key names generated by the job.

* **Secret Name**: `keycloak-secret`
* **Client ID Key**: `clientId`
* **Client Secret Key**: `client-secret`

Do not define or reference custom secret names (e.g., `roundcube-oauth-secret` or `immich-keycloak-secret`) in deployment templates.
